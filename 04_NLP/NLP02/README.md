## 🐛 핵심 버그 수정 — Pre-LN final_norm

### 문제 발생 원인

```
① 모델이 Pre-LayerNorm 구조
   → sub-layer 안에서는 LayerNorm이 입력을 정규화해줌
   → BUT Encoder 스택 전체가 끝난 마지막 출력에는 정규화 누락!
        ↓
② residual connection이 두 번 누적
   → 출력의 norm이 폭주한 상태로 cross-attention의 Key로 입력됨
        ↓
③ Q × K^T 값이 너무 커짐
   → softmax가 한 위치에만 100% attention이 박혀버리는 상태
        ↓
④ 모드 붕괴(Mode Collapse) 발생
   → 출력 결과가 단조롭게 반복되는 현상
```

> 💬 **모드 붕괴(Mode Collapse)** 란?  
> 모델이 훈련 데이터의 다양한 패턴을 무시하고 극히 제한적이거나 단조로운 결과물만 반복해서 생성하는 오류  
> — *김민욱 검색 붙임*

### 해결 방법

```python
# Encoder/Decoder 스택 끝에 final_norm 한 줄씩 추가
s.enc_final_norm = tf.keras.layers.LayerNormalization(epsilon=1e-6)
s.dec_final_norm = tf.keras.layers.LayerNormalization(epsilon=1e-6)

# call 마지막에 통과
ctx = s.enc_final_norm(ctx)  # ← residual 누적 norm 폭주 방지
x   = s.dec_final_norm(x)    # ← softmax 모드 붕괴 방지 핵심
```

> ⚠️ **핵심 포인트**: 뭔가 대단한 걸 추가한 게 아니라,  
> **이게 그냥 표준 Pre-LayerNorm Transformer 구조**임.  
> 즉, **있어야 할 게 없어서 생긴 일.**

## 💡 회고

### 배운 점

- **Pre-LN vs Post-LN**: 스택 마지막에 `final_norm`이 없으면 residual 누적으로 norm이 폭주하고, softmax가 한 위치에 100% 집중되는 **모드 붕괴**가 발생한다는 것을 코드로 직접 경험했어요.
- **데이터 증강**: Lexical Substitution으로 한정된 챗봇 데이터를 3배로 늘려 루브릭을 충족시키는 방법을 배웠어요.
- **Masked Loss**: `<pad>` 토큰을 제외하고 실제 의미 있는 토큰에 대해서만 Loss를 계산하는 것의 중요성을 이해했어요.
- **Beam Search**: Greedy Decoding보다 더 자연스러운 답변 생성 가능성을 확인했어요.
- **Label Smoothing**: 하드 레이블 대신 소프트 레이블로 일반화 성능을 높이는 방법을 배웠어요.

### 아쉬운 점

- d_model이 368으로 작아 표현력에 한계가 있었어요. 512로 키운 개선 모델에서 Loss가 크게 줄었어요.
- 형태소 분리된 답변 출력이 아쉬워요. 후처리로 형태소를 합치면 더 자연스러울 것 같아요.
